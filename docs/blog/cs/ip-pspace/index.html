Visualization and detailed description of the interactive protocol for TQBF yielding IP = PSPACE

03.04.2022 in Computer Science → Complexity theory, Logic Last modified at: 05.04.2022

In complexity theory, it is well known that IP = PSPACE. In this post I would like to go over the private coin interactive protocol for TQBF typically used to prove that PSPACE is contained in IP and, in that context, present my detailed visualization of the entire protocol together with the Open Source tool I developed to run the protocol and render the animations. I would also like to note that when I was first studying the proof protocol for TQBF, I was a bit frustrated by the fact that most texts lack a precise and detailed description of some subtleties in the proof — this is what also motivated me to try my best to fill in the relevant but missing details in this post.

Shamir’s Theorem

In 1990, Adi Shamir proved that IP⁡=PSPACE⁡\operatorname{IP} = \operatorname{PSPACE}IP=PSPACE. I find this result quite surprising, because it shows that just allowing the verifier to toss coins increases the computational power of polynomially long lasting interactive proof systems drastically, namely from dIP⁡=NP⁡\operatorname{dIP} = \operatorname{NP}dIP=NP to PSPACE⁡\operatorname{PSPACE}PSPACE. It is not hard to see why IP⁡⊆PSPACE⁡\operatorname{IP} \subseteq \operatorname{PSPACE}IP⊆PSPACE — this is the case because we can try running the interactive proof system for all possible random bits the verifier may use and messages the prover could potentially send. After that, it is possible to check whether we were able to find a way to convince the verifier for most possibilities of choosing random bits. More precisely, a PSPACE⁡\operatorname{PSPACE}PSPACE decider for L∈IP⁡L \in \operatorname{IP}L∈IP can be constructed as follows (pseudocode):

c = 0
threshold = ceil(2/3 * (total number of different ways the verifier can toss coins))
for all possible verifier's random bits {
    for all possible message sequences the prover may send during the communication {
        Run the verifier using the current message sequence and random bits;
        if verifier accepted {
            c++;
            break;
        }
    }
}
if c >= threshold {
    accept;
} else {
    reject;
}

So the main challenge of showing IP⁡=PSPACE⁡\operatorname{IP} = \operatorname{PSPACE}IP=PSPACE is actually showing that PSPACE⁡⊆IP⁡\operatorname{PSPACE} \subseteq \operatorname{IP}PSPACE⊆IP. Since IP⁡\operatorname{IP}IP is closed under polynomial time Karp reductions and TQBF⁡\operatorname{TQBF}TQBF is PSPACE⁡\operatorname{PSPACE}PSPACE-complete, it suffices to show that TQBF⁡∈IP⁡\operatorname{TQBF} \in \operatorname{IP}TQBF∈IP. There are multiple ways of proving it — I will generally follow Shamir’s approach and use a trick called linearization, which was introduced by Shen in this paper.

The main tool used in the proof is arithmetization, that is, the representation of QBF sentences using polynomials. Refinement of arithmetization

Before we can make use of the representation of QBF’s as polynomials in the interactive proof protocol, we need to ensure that they satisfy some “nice properties”. In this section, I will define such properties and show how we can ensure that they indeed hold. To make the notation introduced above more robust and easier to work with, we can first define the following functions that take a multivariate polynomial ppp as input and produce another polynomial: ∃xi(p)(x1,…,xn):=p(x1,…,xi−1,0,xi+1,…,xn)+p(x1,…,xi−1,1,xi+1,…,xn)∀xi(p)(x1,…,xn):=p(x1,…,xi−1,0,xi+1,…,xn)⋅p(x1,…,xi−1,1,xi+1,…,xn)Qxi(p):={∃xi(p)if xi is existentially quantified∀xi(p)if xi is universally quantified\begin{aligned} \exists_{x_i}(p)(x_1, \dots, x_n) &:= p(x_1, \dots, x_{i-1}, 0, x_{i+1}, \dots, x_n) + p(x_1, \dots, x_{i-1}, 1, x_{i+1}, \dots, x_n) \\ \forall_{x_i}(p)(x_1, \dots, x_n) &:= p(x_1, \dots, x_{i-1}, 0, x_{i+1}, \dots, x_n) \cdot p(x_1, \dots, x_{i-1}, 1, x_{i+1}, \dots, x_n) \\ Q_{x_i}(p) &:= \begin{cases} \exists_{x_i}(p) & \text{if } x_i \text{ is existentially quantified} \\ \forall_{x_i}(p) & \text{if } x_i \text{ is universally quantified} \end{cases} \end{aligned}∃xi​​(p)(x1​,…,xn​)∀xi​​(p)(x1​,…,xn​)Qxi​​(p)​:=p(x1​,…,xi−1​,0,xi+1​,…,xn​)+p(x1​,…,xi−1​,1,xi+1​,…,xn​):=p(x1​,…,xi−1​,0,xi+1​,…,xn​)⋅p(x1​,…,xi−1​,1,xi+1​,…,xn​):={∃xi​​(p)∀xi​​(p)​if xi​ is existentially quantifiedif xi​ is universally quantified​​

The main equivalence we got above (as a result of arithmetization) can now be rewritten as follows: φ∈TQBF⁡⇔Qx1Qx2…QxnPφ≠0\varphi \in \operatorname{TQBF} \Leftrightarrow Q_{x_1} Q_{x_2} \dots Q_{x_n} P_{\varphi} \neq 0φ∈TQBF⇔Qx1​​Qx2​​…Qxn​​Pφ​​=0

From now on, I will call QxiQ_{x_i}Qxi​​ a proof operator. The intuitive idea behind the interactive proof protocol we want to build is to check Qx1Qx2…QxnPφ≠0Q_{x_1} Q_{x_2} \dots Q_{x_n}P_{\varphi} \neq 0Qx1​​Qx2​​…Qxn​​Pφ​​=0 by iteratively considering proof operators from left to right and in each step asking the prover to describe how the “behavior changes” using a univariate polynomial. As we will see below, this can be done in such a way so that the verifier can easily check whether what the prover sent “makes somewhat sense”, and if yes, then if the prover indeed lied, he will be forced with high probability to lie again and again, which leads to a high probability of the verifier finding out that the prover is dishonest at some stage — if the verifier catches the prover, he dismisses the alleged proof.

Linearization

For the polynomial-time verifier to be able to work properly with polynomials, we (will) need to ensure that they have low degree, i.e., the degree must itself be bounded by a polynomial in the size of φ\varphiφ. Note that the degree of PφP_{\varphi}Pφ​ is low, because deg⁡(Pφ)≤n⋅m\deg(P_{\varphi}) \le n \cdot mdeg(Pφ​)≤n⋅m, where mmm is the amount of clauses. While ∃xi\exists_{x_i}∃xi​​ adds polynomials which doesn’t increase their degree, ∀xi\forall_{x_i}∀xi​​ multiplies polynomials meaning that the degree can get doubled in the worst case. If this happens in every step, then the degree of the entire arithmetization may get exponential, which is unacceptable. To resolve this issue, we can introduce a linearity operator LxiL_{x_i}Lxi​​ defined as follows: Lxi(p)(x1,…,xn):=xi⋅p(x1,⋯xi−1,1,xi+1,…,xn)+(1−xi)⋅p(x1,⋯xi−1,0,xi+1,…,xn)\begin{aligned} L_{x_i}(p)(x_1, \dots, x_n) &:= x_i \cdot p(x_1, \dotsm x_{i-1}, 1, x_{i+1}, \dots, x_n) \\ &+ (1 - x_i) \cdot p(x_1, \dotsm x_{i-1}, 0, x_{i+1}, \dots, x_n) \end{aligned}Lxi​​(p)(x1​,…,xn​)​:=xi​⋅p(x1​,⋯xi−1​,1,xi+1​,…,xn​)+(1−xi​)⋅p(x1​,⋯xi−1​,0,xi+1​,…,xn​)​

The idea is the following: LxiL_{x_i}Lxi​​ outputs a polynomial that agrees with ppp on all x1,…,xn∈{0,1}x_1, \dots, x_n \in \{0,1\}x1​,…,xn​∈{0,1} such that the resulting polynomial is linear in xix_ixi​. Modular arithmetic

In the previous section, we’ve ensured that the intermediate polynomials all have low degree. However, there is another problem — we still may not be able to evaluate such low degree polynomials in polynomial time, because the value can get exponentially large with respect to the previous value, at every proof operator, meaning that the tight upper bound for the value of the entire polynomial is O(2(2n))O(2^{(2^n)})O(2(2n)). We can resolve this issue by picking a prime number ppp and doing all computations modulo ppp, that is, in the Fp\mathbb{F}_pFp​ field. This doesn’t break any equivalence we have shown above, since there is a constant d∈Nd \in \mathbb{N}d∈N such that Qx1Qx2…QxnPφ≠0⇔⇔There is a prime p∈O(2nd) such that Qx1Qx2…QxnPφ≢0mod  p\begin{aligned} &Q_{x_1} Q_{x_2} \dots Q_{x_n} P_{\varphi} \neq 0 \Leftrightarrow \\ &\Leftrightarrow\text{There is a prime } p \in O(2^{n^d}) \text{ such that } Q_{x_1} Q_{x_2} \dots Q_{x_n} P_{\varphi} \not\equiv 0 \mod p \end{aligned}​Qx1​​Qx2​​…Qxn​​Pφ​​=0⇔⇔There is a prime p∈O(2nd) such that Qx1​​Qx2​​…Qxn​​Pφ​​≡0modp​

holds. This is not hard to show using number theory — see Theorem 4 of Shamir’s paper for the details. Moreover, we can assume that p>2np > 2^np>2n — this will be important for the correctness of the protocol that I will discuss at the end of this post.

Note that, as stated above, if Qx1Qx2…QxnPφ≠0Q_{x_1} Q_{x_2} \dots Q_{x_n} P_{\varphi} \neq 0Qx1​​Qx2​​…Qxn​​Pφ​​=0, then the same statement will be true modulo at least one prime number ppp that can be encoded using polynomially many bits. This does not mean that any prime p>2np > 2^np>2n will work — the problem is that in the arithmetization polynomial we simulate ∃\exists∃ quantifiers using a sum, but two non-zero values may become zero modulo ppp once added together. For example, consider the formula φ=∃x∀y∃z∃u∃w:(x∨z‾)∧(z‾∨u‾∨w‾)∧(x‾∨y‾∨u‾∨w‾)∧(y‾∨z‾)\varphi = \exists x \forall y \exists z \exists u \exists w : (x \vee \overline{z}) \wedge (\overline{z} \vee \overline{u} \vee \overline{w}) \wedge (\overline{x} \vee \overline{y} \vee \overline{u} \vee \overline{w}) \wedge (\overline{y} \vee \overline{z})φ=∃x∀y∃z∃u∃w:(x∨z)∧(z∨u∨w)∧(x∨y​∨u∨w)∧(y​∨z)

with n=5n = 5n=5 variables. We could be tempted to just choose the first prime p>2n=32p > 2^n = 32p>2n=32, which is p=37p = 37p=37. Calculating the value of the arithmetizing polynomial gives us: ∀y∃z∃u∃wPφ=5⋅x+16∃x∀y∃z∃u∃wPφ=(5⋅0+16)+(5⋅1+16)=16+21=37=0\begin{aligned} \forall_y \exists_z \exists_u \exists_w P_{\varphi} &= 5 \cdot x + 16 \\ \exists_x \forall_y \exists_z \exists_u \exists_w P_{\varphi} &= (5 \cdot 0 + 16) + (5 \cdot 1 + 16) \\ &= 16 + 21 = 37 = 0 \end{aligned}∀y​∃z​∃u​∃w​Pφ​∃x​∀y​∃z​∃u​∃w​Pφ​​=5⋅x+16=(5⋅0+16)+(5⋅1+16)=16+21=37=0​

However, although ∃x∀y∃z∃u∃wPφ\exists_x \forall_y \exists_z \exists_u \exists_w P_{\varphi}∃x​∀y​∃z​∃u​∃w​Pφ​ suggests that φ∉TQBF⁡\varphi \notin \operatorname{TQBF}φ∈/​TQBF, φ\varphiφ is actually a true sentence. This example illustrates why we need to be careful when choosing ppp.

Interactive protocol for TQBF

As mentioned earlier, the vague idea behind the protocol is to go over the operators from left to right and ask the the prover to describe “what has changed” using a polynomial. Here, we assume without loss of generality that the variable of the first proof operator is x1x_1x1​ (rename variables if this is not the case). Correctness of the protocol

Finally, to make the description of the protocol in this post complete, I would like to explain in my own words why the protocol defined above is correct. The completeness property of IP⁡\operatorname{IP}IP holds obviously — if φ∈TQBF⁡\varphi \in \operatorname{TQBF}φ∈TQBF, then an honest prover will be able to convince the verifier no matter how the verifier tosses coins.

For the soundness analysis, we will first show the following lemma:

Lemma: Let ggg be a polynomial, c,a1,…,ak∈Fpc, a_1, \dots, a_k \in \mathbb{F}_pc,a1​,…,ak​∈Fp​ and suppose that we are checking whether g(a1,…,ak)=cg(a_1, \dots, a_k) = cg(a1​,…,ak​)=c using the operator composition check protocol, but actually g(a1,…,ak)≠cg(a_1, \dots, a_k) \neq cg(a1​,…,ak​)​=c. Then the probability of either

  • the verifier rejecting the proof in that very round (at step 4), or
  • the verifier forcing the prover to continue lying in the next round, that is, re-running the protocol to verify s(a)=g′(a,a2,…,ak)s(a) = g'(a, a_2, \dots, a_k)s(a)=g′(a,a2​,…,ak​) when it is actually the case that s(a)≠g′(a,a2,…,ak)s(a) \neq g'(a, a_2, \dots, a_k)s(a)​=g′(a,a2​,…,ak​)

is at least 1−d/p1 - d/p1−d/p, where d=deg⁡(h(x1))d = \deg(h(x_1))d=deg(h(x1​)).

Proof: If the prover sends s=hs = hs=h, then, depending on the current proof operator, one of the checks in step 4 will fail and the verifier will therefore reject with probability 1≥1−d/p1 \ge 1-d/p1≥1−d/p. On the other hand, if the prover sends s≠hs \neq hs​=h, then the polynomial s(x1)−h(x1)≠0s(x_1) - h(x_1) \neq 0s(x1​)−h(x1​)​=0 has degree at most ddd. This means that s−hs - hs−h has at most ddd roots, i.e., there exist at most ddd values a∈Fpa \in \mathbb{F}_pa∈Fp​ such that s(a)=h(a)s(a) = h(a)s(a)=h(a). In other words, Pr⁡[s(a)=h(a)]≤d/p\Pr[s(a) = h(a)] \le d/pPr[s(a)=h(a)]≤d/p. Since the verifier uses the value c=s(a)c = s(a)c=s(a) in the next round, it holds that Pr⁡[s(a)≠g′(a,a2,…,ak)]=Pr⁡[s(a)≠h(a)]≥1−d/p\Pr[s(a) \neq g'(a, a_2, \dots, a_k)] = \Pr[s(a) \neq h(a)] \ge 1 - d/pPr[s(a)​=g′(a,a2​,…,ak​)]=Pr[s(a)​=h(a)]≥1−d/p

This completes the proof of the lemma.

Now suppose that φ∉TQBF⁡\varphi \notin \operatorname{TQBF}φ∈/​TQBF. Then it is clear that the probability of the verifier not being fooled by the prover is at least the probability of the prover being forced to lie again and again, until he is either caught at some (possibly last) round. By applying the above lemma inductively, we get: Pr⁡[Verifier rejects or prover continues lying after k rounds]≥(1−dp)k\Pr[\text{Verifier rejects or prover continues lying after } k \text{ rounds}] \ge \Big(1 - \frac{d}{p}\Big)^kPr[Verifier rejects or prover continues lying after k rounds]≥(1−pd​)k

Since k≤2⋅n2∈O(n2)k \le 2 \cdot n^2 \in O(n^2)k≤2⋅n2∈O(n2), p>2np > 2^np>2n and d≤cnc+cd \le cn^c+cd≤cnc+c for some constant ccc, it holds that Pr⁡[Verifier rejects]≥(1−dp)2n2≥(1−cnc+c2n)2n2\Pr[\text{Verifier rejects}] \ge \Big(1 - \frac{d}{p}\Big)^{2n^2} \ge \Big(1 - \frac{cn^c+c}{2^n}\Big)^{2n^2}Pr[Verifier rejects]≥(1−pd​)2n2≥(1−2ncnc+c​)2n2

We can calculate the limit of this probability using L’Hôpital’s rule: lim inf⁡n→∞Pr⁡[Verifier rejects]≥lim⁡n→∞(1−cnc+c2n)2n2=1\liminf_{n \to \infty}{\Pr[\text{Verifier rejects}]} \ge \lim_{n \to \infty}{\Big(1 - \frac{cn^c+c}{2^n}\Big)^{2n^2}} = 1n→∞liminf​Pr[Verifier rejects]≥n→∞lim​(1−2ncnc+c​)2n2=1

By the definition of the limit, this means that for any 0<ε≤10 < \varepsilon \le 10<ε≤1 there exists a fixed n∈Nn \in \mathbb{N}n∈N such that for all QBF sentences φ\varphiφ with at least nnn variables the probability of the verifier rejecting φ∉TQBF⁡\varphi \notin \operatorname{TQBF}φ∈/​TQBF is at least 1−ε1 - \varepsilon1−ε. This in particular implies the soundness requirement of the IP⁡\operatorname{IP}IP complexity class, thus completing the proof of the correctness of the protocol.

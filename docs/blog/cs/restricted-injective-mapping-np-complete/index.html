Finding an injective mapping with restrictions for values is NP-Complete

07.08.2021 in Computer Science → Research, Complexity theory Last modified at: 09.08.2021

This post is about an interesting problem I recently tried to solve efficiently but ended up proving its NP\mathcal{NP}NP-completeness. Informally, the problem is to find an injective mapping between two finite sets with certain, statically defined properties for values. For example, if A={a,b,c,d}A = \{a, b, c, d\}A={a,b,c,d} and B={1,2,3,4,5}B = \{1, 2, 3, 4, 5\}B={1,2,3,4,5}, we may want to find an injective mapping f:A→Bf : A \rightarrow Bf:A→B with restrictions f(a)∈{1,3,4,5}f(a) \in \{1, 3, 4, 5\}f(a)∈{1,3,4,5}, f(b)∈{3,4,5}f(b) \in \{3, 4, 5\}f(b)∈{3,4,5}, f(c)∈{1,2,4,5}f(c) \in \{1, 2, 4, 5\}f(c)∈{1,2,4,5}, f(d)∈{1,2,5}f(d) \in \{1, 2, 5\}f(d)∈{1,2,5}, f(e)∈{1,2,3,4}f(e) \in \{1, 2, 3, 4\}f(e)∈{1,2,3,4}. I find this problem of particular interest, because its nature is in my opinion very different compared to well-known NP\mathcal{NP}NP-complete problems proven to be so by Karp. In this post we will visualize and formalize this problem, prove its NP\mathcal{NP}NP-completeness and, finally, derive some interesting corollaries.

The problem

The above intuitive description of the problem can be equivalently formalized as follows:

Name: Restricted injective mapping

Input: A set AAA as well as a collection S={A1,…,An}S = \{A_1, \dots, A_n\}S={A1​,…,An​} of subsets of AAA, i.e. Ai⊆AA_i \subseteq AAi​⊆A for all 1≤i≤n1 \le i \le n1≤i≤n.

Question: ∃(a1,…,an)∈A1×⋯×An\exists (a_1, \dots, a_n) \in A_1 \times \dots \times A_n∃(a1​,…,an​)∈A1​×⋯×An​ such that ai≠aja_i \neq a_jai​​=aj​ for all i≠j,1≤i,j≤ni \neq j, 1 \le i,j \le ni​=j,1≤i,j≤n?

This definition is equivalent to the brief description of the problem at the beginning of this post, since we can only focus on the restrictions in the target set (assuming that the elements in the source set are numbered) and it is possible to define such an injective mapping fff​ if and only if ∃(a1,…,an)∈A1×⋯×An\exists (a_1, \dots, a_n) \in A_1 \times \dots \times A_n∃(a1​,…,an​)∈A1​×⋯×An​​ such that ai≠aja_i \neq a_jai​​=aj​​ for all i≠ji \neq ji​=j​.

A special case of the problem

Every injective mapping f:A→Bf : A \rightarrow Bf:A→B with ∣A∣=∣B∣∈N0\st A \st = \st B \st \in \mathbb{N}_0∣A∣=∣B∣∈N0​ is also surjective and bijective. This leads us to an interesting special case of Restricted injective mapping, where we just add an additional constraint, that the amount of subsets in the collection SSS must be equal to the cardinality of AAA:

Name: Restricted bijective mapping

Input: A set AAA of cardinality ∣A∣=n\st A \st = n∣A∣=n and a collection of nnn subsets S={A1,…,An}S = \{A_1, \dots, A_n\}S={A1​,…,An​} of AAA, i.e. Ai⊆AA_i \subseteq AAi​⊆A for all 1≤i≤n1 \le i \le n1≤i≤n.

Question: ∃(a1,…,an)∈A1×⋯×An\exists (a_1, \dots, a_n) \in A_1 \times \dots \times A_n∃(a1​,…,an​)∈A1​×⋯×An​ such that ai≠aja_i \neq a_jai​​=aj​ for all i≠j,1≤i,j≤ni \neq j, 1 \le i,j \le ni​=j,1≤i,j≤n?

In other words and in light of the above visualization of an example instance of Restricted injective mapping with a table, this special case has an additional requirement that there are exactly as many rows as columns.

Proof idea

In order to prove NP\mathcal{NP}NP​​​-hardness of the Restricted bijective mapping problem above, we need to construct a reduction from an already-proven NP\mathcal{NP}NP​​​-complete problem. This reduction must somehow express the constraints of the original problem with the collection of subsets SSS​​​. Many NP\mathcal{NP}NP​​​​-complete problems are graph-related and in graphs, a path with disjoint vertices seems to have very similar constraints compared to our problem. It may therefore be a good idea to let AAA be the set of nodes VVV in an undirected graph G=(V,E)G = (V, E)G=(V,E) where, without loss of generality, V={1,…,n}V = \{1, \dots, n\}V={1,…,n} and define AiA_iAi​ for 1≤i≤n1 \le i \le n1≤i≤n as follows: Ai:={v∣{i,v}∈E}A_i := \{v \mid \{i, v\} \in E\}Ai​:={v∣{i,v}∈E}

The Hamiltonian circuit problem is a well-known NP\mathcal{NP}NP​-complete problem, which is formally defined as follows:

Name: HC

Input: Undirected graph G=(V,E)G = (V, E)G=(V,E)

Question: Does there exist a hamiltonian circuit, i.e. a cycle that visits every vertex exactly once?

If we prove NP\mathcal{NP}NP-hardness of HC with a reduction fff that cannot produce graphs with two or more vertex-disjoint cycles such that every node is in exactly one such cycle, and then prove NP\mathcal{NP}NP-hardness of Restricted bijective mapping with a reduction ggg that assumes that the input graph cannot be partitioned in vertex-disjoint cycles as mentioned above, then the composite reduction g∘fg \circ fg∘f will prove NP\mathcal{NP}NP-hardness of our problem. For a formal proof I will therefore need to reiterate a variant of the well-known proof of NP\mathcal{NP}NP​-completeness of HC emphasizing the fact that only graphs with the above mentioned constraint are being produced.

This problem is obviously in NP\mathcal{NP}NP, because we can use a sequence of vertices as a certificate that can be verified in polynomial time by making sure that:

  • this sequence of vertices is a path with disjoint vertices;
  • the last vertex is connected with the first one;
  • every node of the graph is in the sequence;

Corollaries

NP-completeness of finding an injective mapping

Restricted injective mapping is obviously in NP\mathcal{NP}NP​.

The NP\mathcal{NP}NP​-hardness of the present problem follows directly from the NP\mathcal{NP}NP-hardness of Restricted bijective mapping as we can construct a trivial reduction from the latter problem that just passes the instance without any changes.
